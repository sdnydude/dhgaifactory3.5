# DHG Docs Expansion Guide

How to add a new DHG project to the documentation hub.

> **Architecture note:** this site uses a SINGLE `@docusaurus/preset-classic` docs
> instance (`docusaurus.config.ts` ŌåÆ `presets`), with `projects/` mounted at the
> site root and one sidebar file (`sidebars.ts`) containing a collapsed category
> per project. There are NO per-project plugin instances and NO per-project
> sidebar files. (An earlier multi-instance design was abandoned June 2026.)


## 2. Register the project (4 places, all required)

1. **Sidebar** ŌĆö add a category block to `docs-site/sidebars.ts`:

```ts
{
  type: 'category',
  label: 'Project Name',
  collapsed: true,
  items: [{type: 'autogenerated', dirName: '<project-name>'}],
},
```

2. **Homepage card** ŌĆö add an entry to the `projects` array in
   `docs-site/src/pages/index.tsx`.

3. **Footer link** ŌĆö add to the `Projects` footer column in
   `docs-site/docusaurus.config.ts`.

4. **(Optional) Navbar Tools dropdown** ŌĆö if the project has a live web UI,
   add it to the `Tools` dropdown in `docusaurus.config.ts` and a service card
   in `index.tsx` (Apps / Observability / Data & Admin / Labs).

> Skipping steps 2ŌĆō3 is how the homepage went stale in the past. All four
> registrations are part of "adding a project."

## 3. Ingest into registry

Run the ingest script to populate the semantic search index:

```bash
cd /home/swebber64/DHG/aifactory3.5/dhgaifactory3.5/registry
python3 doc_ingest.py \
  --project <project-name> \
  --docs-dir /path/to/docs \
  --sweep
```

## 4. Add to CI/CD

Add a step to `.github/workflows/deploy-docs.yml` in the relevant project repo:

```yaml
- name: Ingest <project> docs
  run: |
    python3 /home/swebber64/DHG/aifactory3.5/dhgaifactory3.5/registry/doc_ingest.py \
      --project <project-name> \
      --docs-dir <path-to-docs> \
      --sweep
```

## 5. Build and verify

`build/` is bind-mounted into the nginx container. `npm run build` recreates the
`build/` directory (new inode), which **orphans the container's bind mount** ŌĆö
nginx then serves 403. You MUST restart dhg-docs after every build:

```bash
cd /home/swebber64/DHG/aifactory3.5/dhgaifactory3.5/docs-site
npm run build
docker restart dhg-docs      # required ŌĆö remounts build/; skipping it ŌåÆ 403
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8017/<project-name>/getting-started/
```

dhg-docs is now a compose service (`docker-compose.override.yml`), on
`dhgaifactory35_dhg-network` so it can proxy `/api/` to `dhg-registry-api`.

Note: `onBrokenLinks` is set to `throw` ŌĆö a broken internal link fails the
build. Fix the link; do not downgrade the setting.

## Key files

| File | Purpose |
|------|---------|
| `docs-site/docusaurus.config.ts` | Single-instance preset config, navbar, footer |
| `docs-site/sidebars.ts` | ONE sidebar ŌĆö category per project |
| `docs-site/src/pages/index.tsx` | Homepage ŌĆö renders the Patchbay component |
| `docs-site/src/components/Patchbay/services.ts` | Rack/tape/offsite data ŌĆö edit here to add a homepage card. `statusKey` must match `registry/patchbay_service.py` SERVICES |
| `docs-site/src/components/Patchbay/index.tsx` | Patchbay UI: LEDs, filter, Talkback SSE client |
| `docs-site/projects/<project>/` | Markdown content |
| `registry/doc_ingest.py` | Chunking + ingest script |
| `registry/doc_pages_endpoints.py` | API: upsert, bulk, search, delete |
| `registry/talkback_endpoints.py` | Talkback assistant: SSE RAG over doc_pages |
| `registry/patchbay_endpoints.py` | Status LEDs: TCP probes of the service map |
| `.github/workflows/deploy-docs.yml` | CI/CD pipeline |
