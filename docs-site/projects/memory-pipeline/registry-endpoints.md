# Registry API Endpoints

The DHG Registry (`dhg-registry-api` on port 8011) provides the storage and search layer for all captured intelligence. Built with FastAPI + PostgreSQL + pgvector.

## Endpoint Groups

### Unified KB Search

The primary recall interface. Searches across all sources with hybrid semantic + full-text ranking (Reciprocal Rank Fusion).

```
POST /api/kb/search
```

```json
{
  "query": "how was the auth system built?",
  "project_name": "portage",
  "sources": ["decisions", "insights", "ship_sessions"],
  "limit": 10
}
```

Sources: `docs`, `insights`, `decisions`, `ship_sessions`, `corrections`, `bug_fixes`, `deferred_items`, `agent_sessions`, `dev_changelog`

### Capture Endpoints

Each data type has its own endpoint module:

| Endpoint | Module | Table | Key Fields |
|----------|--------|-------|------------|
| `POST /api/insights` | `insights_endpoints.py` | `insights` | tldr, insight_statement, category, project_name |
| `POST /api/decision-logs` | `decision_logs_endpoints.py` | `decision_logs` | title, choice, alternatives_rejected, rationale, domain |
| `POST /api/corrections` | `corrections_endpoints.py` | `corrections` | category, user_message, context, claude_action |
| `POST /api/bug-fixes` | `bug_fixes_endpoints.py` | `bug_fixes` | tldr, symptom, root_cause, fix_applied, severity |
| `POST /api/ship-sessions` | `ship_sessions_endpoints.py` | `ship_sessions` | feature, approach, status, pr_url, branch |
| `POST /api/deferred-items` | `deferred_items_endpoints.py` | `deferred_items` | title, description, reason, priority, category |
| `POST /api/test-coverage-changes` | `test_coverage_endpoints.py` | `test_coverage_changes` | title, test_count_before, test_count_after, delta |

### Documentation Search

Separate from KB search — searches only indexed Docusaurus doc pages:

```
POST /api/doc-pages/search
GET  /api/doc-pages?project_name=portage&limit=20
```

## Idempotency

All capture endpoints use upsert logic to prevent duplicates on re-capture:

- **Insights**: deduplicate by `tldr` hash
- **Decisions**: deduplicate by `title` hash
- **Ship sessions**: deduplicate by `feature + branch`
- **Corrections**: deduplicate by `user_message` hash
- **Bug fixes**: deduplicate by `tldr` hash
- **Deferred items**: deduplicate by `title` hash

## Database

PostgreSQL 15 with pgvector extension. Key tables for the memory pipeline:

| Table | Records | Purpose |
|-------|---------|---------|
| `insights` | 50+ | AI-captured insight blocks |
| `decision_logs` | 12+ | Architectural decisions with alternatives |
| `corrections` | 10+ | User corrections to Claude behavior |
| `bug_fixes` | 15+ | Root cause analyses |
| `ship_sessions` | 37+ | /ship workflow records |
| `deferred_items` | 15+ | Work discovered but intentionally deferred |
| `test_coverage_changes` | 5+ | Test count deltas |
| `doc_pages` | 319+ | Docusaurus doc chunks with embeddings |
| `agent_sessions` | 100+ | Claude Code session summaries |

Embeddings generated by `nomic-embed-text` via Ollama (port 11434) through the session-logger service (port 8009).
