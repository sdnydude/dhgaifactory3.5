# Debug Protocol

When encountering any bug, error, or unexpected behavior:

1. **PAUSE** — State the problem clearly before touching code
2. **RESEARCH** — Check logs, recent changes, and error messages first
3. **HYPOTHESIZE** — Form 2-3 ranked hypotheses before attempting fixes
4. **PLAN** — Present hypotheses to user for approval before acting
5. **FIX** — One fix attempt per hypothesis only. If it fails, move to next hypothesis.
6. **VERIFY** — Run verification commands and show output as proof
7. **DOCUMENT** — State root cause, fix applied, and prevention measure

## Rules
- Never repeat the same failed fix
- Never change multiple things at once
- If 3 hypotheses fail, escalate — step back and reassess fundamentals
- Log every error and its resolution for future reference
