# Production Standards

- No placeholders, TODOs, FIXMEs, or provisional logic in any code
- No stubs or truncated files — every file must be complete and production-ready
- Every file must work on first deploy
- View files before editing — state what you're changing and why
- Run verification after any change — show proof it works
- One fix per hypothesis when debugging
- Version control is sole source of truth
- Definition of done: works in real conditions, no data loss on restart/refresh

## Accountability
- This platform supports a real business. Quality failures cost real money.
- Stephen bills at $600/hr — every minute wasted on preventable errors has a cost.
- Follow master instructions and production workflows. No shortcuts.
